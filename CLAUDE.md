## Commands

```bash
npm run dev          # Start dev server (port 3333)
npm run build        # Production build
npm run preview      # Preview production build
npm run ingest       # Local: scan assets, compress, generate projects.ts
node scripts/upload-to-r2.mjs  # One-time: upload local assets to R2
```

## Environment

Required in `.env` (and Vercel env vars for prod):

```
ADMIN_PASSWORD=<strong password>
R2_ACCOUNT_ID=<cloudflare account id>
R2_ACCESS_KEY_ID=<r2 access key>
R2_SECRET_ACCESS_KEY=<r2 secret>
R2_ENDPOINT=https://<account-id>.r2.cloudflarestorage.com
R2_BUCKET_NAME=lyned-assets
R2_PUBLIC_URL=https://pub-<id>.r2.dev
```

## Stack

Nuxt 4.4 / Vue 3.5 / Tailwind CSS 4 (Vite plugin) / TypeScript / WebGL2 / Cloudflare R2 / aws4fetch

SPA mode (`ssr: false`) ‚Äî WebGL and heavy client-side rendering make SSR impractical. Deployed on Vercel.

## Architecture

Interactive portfolio with WebGL dithering background, three view modes, admin CMS backed by Cloudflare R2.

### Infinite Canvas Tiling (`SandboxCanvas.vue`)

Canvas bounds computed from project positions. Drag offset wrapped via modulo. 3x3 tile grid rendered, only viewport-overlapping tiles kept (`activeTiles`). Scoped slot provides `isVisible(project)` for lazy loading. Receives `projects` as prop (not imported statically).

Drag: unified `PointerEvent`, momentum with friction (0.92), lerp (0.15). rAF loop auto-stops when converged, restarts on `ensureAnimating()`.

### Three View Modes

- **Canvas** (default): infinite draggable sandbox with `SandboxCanvas`
- **Masonry**: CSS `columns` responsive grid with staggered reveal animation
- **List**: vertical table rows with slide-in animation

Switched via `ViewToggle` (bottom right). `<Transition>` with directional slide (left/right based on order). `open.mp3` plays on switch.

### Video Lazy Loading (`ProjectCard.vue`)

`watch(visible, ..., { immediate: true })`: visible ‚Üí mount video, hidden ‚Üí pause + 10s unload timer. Thumbnail behind video as fallback. Only rendered for video projects (images use single `<img>`).

### R2 Module (`server/lib/r2.ts`)

Uses `aws4fetch` (3KB) instead of `@aws-sdk/client-s3` (3MB) to avoid Vite SSR IPC crashes. Lazy singleton client. All API routes import dynamically: `await import('~~/server/lib/r2')`. Note: `~~` = rootDir (not `~` which is `app/` in Nuxt 4).

### Auth (`server/utils/admin-auth.ts`)

Session tokens via `crypto.randomBytes`, stored in-memory Map. Cookie `admin-session` (httpOnly) + `x-admin-token` header fallback. `crypto.timingSafeEqual` for comparisons. Rate limiting: 5 attempts ‚Üí 15min lockout per IP.

### Admin (`app/pages/admin.vue`)

`definePageMeta({ layout: false })`. Upload drag & drop with client-side WebP compression (Canvas API). Preview modal reuses the tech frame design. MKV/MOV rejected (MP4/WebM only). Magic bytes validation on uploads. 50MB size limit.

### Preloader (`Preloader.vue`)

Click-to-enter gate (required for audio autoplay). 5-phase boot sequence (~6.5s): corner brackets ‚Üí logo reveal + glitch ‚Üí progress bar with accelerating click.mp3 ticks ‚Üí peripheral data ‚Üí completion. Timers cleaned up on unmount.

### Sound (`useSound.ts`)

Module-scoped `muted` ref + `cache` Map + `preloaded` flag. `forceClick()` bypasses mute (used by SoundToggle). SSR returns no-ops.

## Key Patterns

- **Clip-path corners** are the design language: 8-10px on buttons, 12px on cards, 20px on modal
- **`v-show` for filtering**, not `v-if` ‚Äî preserves layout and avoids media reload
- **Canvas disables** when modal open via `:disabled` prop
- **Cursor swap intentional**: `.cursor-grab` uses closed-hand, `.cursor-grabbing` uses open-hand (shows current state)
- **`~~/server/lib/r2`** for all R2 imports in server routes (dynamic import to avoid bundling at boot)

## Gotchas

- `ssr: false` ‚Äî the site is a SPA. No server-side rendering of pages.
- `v-show` filtered cards still play video in background (intentional).
- The rAF loop in SandboxCanvas stops when idle. Call `ensureAnimating()` if adding reactive dependencies.
- WebGL `createProgram` can return null ‚Äî always guard.
- `useSound()` calls `onMounted` internally ‚Äî only call inside `setup()`.
- `ADMIN_PASSWORD` has no fallback ‚Äî server throws if env var is missing.
- R2 errors are logged server-side but return generic messages to client.
- `FilterDropdown` receives live projects via prop ‚Äî don't import from `data/projects.ts` (that file is dead code except for the `Project` type).
